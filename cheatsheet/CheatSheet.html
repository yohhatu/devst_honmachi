# 初期設定 # ユーザネームとメールアドレスの登録 $ git config --global user.name "My Name" $ git config --global user.email my.name@example.com "My Name"の"my.name@example.com"には自分のメールアドレスを入力します. これらの情報を登録しておかないとログに名前が残らないので, 誰が更新をしたかわからなくなります. # ログを残す際のエディタ $ git config --global core.editor '(エディタ)' コミットを残す際のエディタを設定できます. 例えばvimにしたい場合は, $ git config --global core.editor 'vim' でOKです. # コンソールの色つき表示 $ git config --global color.diff auto $ git config --global color.status auto $ git config --global color.branch auto で, コンソールの表示に色付けできます. # エイリアス $ git config --global alias.glog 'log --graph --all --color --pretty='%x09%h %cn%x09%s %Cred%d%Creset' ログを気持ちよく(?)見れます. # 代表的なコマンド # status $ git status 現在の状況を表示します. 例えば, 変更したにも関わらずaddしていないファイルが存在するなら, $ git status # On branch master # Changes not staged for commit: # (use "git add ..." to update what will be committed) # (use "git checkout -- ..." to discard changes in working directory) # # modified: CheatSheet.html # no changes added to commit (use "git add" and/or "git commit -a") このようになります. # add $ git add . addします. addの後に"."とすると, 変更された全てのファイルをaddします. # commit $ git commit コミットします. コミットログを入力しないとコミットできません. # checkout $ git checkout チェックアウトします. # pullの流れ $ git pull --rebase conflict発生 ｰ> 修正 $ git rebase --continue 以下, conflictが発生するたびに修正と"git rebase --continue"を繰り返す.
